# Mente 上 GitHub 了：一个能写代码、会记忆、还能跨平台干活的 AI Agent

> 这次我们不是单纯换了个名字，也不是把几个功能拼在一起，而是把一个真正能长期工作的 agent 产品面重新打磨了一遍。

很多 AI 产品第一次上手很惊艳，但一到真实工作里就开始掉链子：只能在一个窗口里聊天，换个平台就断；今天能帮你，明天又不记得；能回答问题，但很难持续执行。

这正是我们这段时间集中优化 Mente 的原因。现在，它已经正式发布到 GitHub，也把产品最关键的几条能力链路梳顺了。

## 它不是“又一个聊天框”，而是一个真正能工作的统一 Agent

Mente 现在对外是统一的产品身份。你在 CLI 里看到的是它，在消息平台里和你对话的是它，对外展示进度和结果的也是它。

但更重要的是，这个统一不是只停留在名字层面。Mente 背后接的是完整执行系统：

- 能写代码，也能调用工具完成自动化任务
- 能在本地跑，也能在 Docker、SSH、Daytona、Modal 等环境里持续执行
- 能在 Telegram、Discord、Slack、WhatsApp、Signal 和命令行之间连续工作
- 能把执行进度重新展示出来，而不是只返回一句“我做完了”

换句话说，Mente 不只是回答你一句话，而是能接住一整段工作。

## 这一轮优化，重点不是“更聪明”，而是“更能交付”

这次我们做的核心优化，集中在真正影响使用体验的地方。

**第一，产品表面统一了。**  
CLI、网关进度、消息平台回复、用户可见名称，现在都统一收口到 Mente，不再让新用户在不同入口里反复猜“这到底是不是同一个东西”。

**第二，底层执行能力保住了。**  
这次对外统一展示，并不意味着能力缩水。复杂编码、工具调用和实际执行，依然走 Codex-backed executor。外层更清晰，底层还是硬的。

**第三，执行过程重新变得可见。**  
很多 agent 的问题不是不会做，而是用户看不见它在做什么。Mente 现在会把对外步骤名整理成可读的 Mente 进度，同时保留底层命令和工具活动明细，便于追踪和排错。

**第四，安装路径更清楚了。**  
GitHub 访客现在就可以直接用一条命令安装：

```bash
curl -fsSL https://raw.githubusercontent.com/chemany/Mente/main/scripts/install.sh | bash
```

npm 路径也已经准备好，后续公开发布后会切到：

```bash
npm install -g mente-agent
mente
```

**第五，配置和运维边界更明确。**  
API key、provider 鉴权、`.env`、`config.yaml`、网关重启这些操作，不再混在普通对话里，而是通过专门的 Mente skill 处理，减少误操作，也更适合长期维护。

这类优化不一定最“炫”，但会直接决定一个 agent 能不能在真实环境里长期用下去。

## Mente 最不一样的，是这 5 个产品特征

### 1. 一个入口，覆盖编码、自动化、消息网关和长期记忆

很多工具只擅长一个点。Mente 想解决的是整条链路：从理解任务，到调工具，到跨会话延续，再到把结果送到你常用的平台。

### 2. 它会记住值得记住的东西

Mente 有持续记忆机制，也会在复杂任务后沉淀技能。你不是每次都从零开始教它，系统会逐步形成稳定做法，并在后续使用中继续迭代。

### 3. 它不被绑在你的笔记本上

你可以把它放在便宜的 VPS 上，也可以挂到 Modal、Daytona 这类接近 serverless 的后端。空闲时休眠，需要时唤醒，更接近一个常驻在线的 worker，而不是“我电脑开着它才活着”。

### 4. 模型和推理服务不是单点绑定

OpenAI、OpenRouter、Nous Portal、NVIDIA NIM、GLM、Kimi、MiniMax、Hugging Face，或者你自己的兼容端点，都可以接进来。通过 `mente model` 就能切，不需要改代码。

### 5. 它可以把任务变成持续运行的系统

定时任务、跨平台投递、子代理并行、基于 RPC 的工具调用，这些能力组合起来，意味着你不只是“问一个问题”，而是在逐步搭一个能长期替你工作的自动化系统。

如果你平时需要写代码、跑工作流、同步消息平台、做日报周报、维护一个常驻在线的 AI 助手，这几个点会比“模型答题更强一点”重要得多。

## 从 GitHub 开始，3 分钟理解怎么上手

如果你第一次接触 Mente，最短路径其实很简单：

1. 在 GitHub 上获取项目并执行安装脚本
2. 运行 `mente`
3. 用 `mente model` 选 provider 和模型
4. 用 `mente tools` 和 `mente gateway` 把工具与消息入口接起来
5. 把一个真实任务交给它，例如定时简报、代码修改、资料整理或跨平台消息处理

几个常用命令如下：

```bash
mente
mente model
mente tools
mente gateway
mente setup
mente doctor
```

如果你来自 OpenClaw，Mente 还提供迁移路径，可以把配置、记忆、技能和部分 secrets 平滑带过来，这会让老用户切换成本低很多。

## 我们为什么现在把它放到 GitHub

因为 Mente 已经不只是内部可用的工具堆栈，而是一个公开用户也能跑起来、也能判断价值的完整产品面。

你现在看到的这版，有统一品牌、有清晰安装路径、有文档、有消息网关、有长期记忆、有技能体系，也保留了足够硬的底层执行能力。它适合继续快速迭代，也值得更早让外部用户直接上手。

如果你一直在找一个不是“只能聊天”、而是真能替你做事的 AI Agent，Mente 现在已经到了值得你亲自试一遍的节点。
